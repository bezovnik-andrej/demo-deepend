Client Feedback Applied (v0.3)

Key decisions from client review that shaped this version.

Removed from scope:
  • Billing / Invoices — handled in HubSpot/CRM. Not part of Norveo UX.
  • Engineers section — not a full management area. Engineer is a stored field (name + email) for signing and contact.
  • Engineer Profile — removed. Just a name and email stored on the admin’s view.
  • Notifications / Alerts — not needed for initial release.
  • Assign Team step in Create Project — removed. Admin creates all projects; no team assignment during creation.
Confirmed decisions:
  • Admin = Engineer for now. Same person, full edit access, same CAD capabilities. Engineer role documented separately for future separation.
  • Single login URL. Role detected server-side. Edge case: shared credentials unlikely to be an issue if no per-seat charging.
  • Full edit workspace — no read-only mode. No separate engineer with restricted access for now.
  • Admin sees Catalogue — yes, since admin is likely the engineer.
  • Settings = gear icon in header, not a sidebar section.
  • Billing — currently access-based (have it or don’t). Paid-per-project is a future consideration, not current.

01 Flow Overview

The Super Admin experience is organized into three layers. The admin is also the engineer for the initial release — one person managing companies, creating projects, and doing the design work.

Layer 1
Admin Back Office
“What needs my attention?”
  • 2 sections: Companies, Projects
  • Each section shows a filterable, searchable list
  • Key actions: add/invite company, create project
  • Global search across all entities
  • Settings via gear icon in header
  • Catalogue access (same as engineer view)
Layer 2
Detail Views
“What is the full picture for this entity?”
  • Project Detail — 8 sub-tabs covering the project lifecycle
  • Company Detail — overview, contacts, linked projects
  • Cross-links: project links to company, company links to projects
  • From Project Detail, admin can open the project in Workspace
Layer 3
Workspace (CAD Tool)
“Design, engineer, and deliver.”
  • Full edit — same CAD tool as the engineer workspace
  • 7 workspace tabs: Design, Summary, Engineering, Estimate, BOM, Deliverables, History
  • Catalogue access via Components panel
  • Breadcrumb: ← Back Office / Projects / Project Name

03 Layer 1: Admin Back Office

The admin lands here after login. Two primary sections in the sidebar. Settings is a gear icon in the header, not a sidebar item.

Navigation

  • Persistent sidebar: Companies, Projects
  • Global search: search bar at the top — searches across companies and projects
  • Breadcrumb: Back Office / Section / Entity Name — visible at every depth level
  • Settings: gear icon in the header → opens settings page (Account, Branding, Integrations)
  • Catalogue: accessible from the sidebar or from within the Workspace

Sections

SectionList View ContentActionsSearch / Filter
Companies Company name, status (active / pending / inactive), project count, contract date Add company (wizard) or invite company Filter by status; search by name
Projects Project name, client, status, config progress, signed engineer (name), last activity Create project Filter by status, client; search by name

Project Statuses

Draft → In Engineering → Ready for Estimate → Sent to Customer → Finalized

Signed Engineer Field

The “engineer” is not a separate role or profile in the system. It is a stored field on the admin’s view:

  • Name and email of the licensed/signing engineer
  • Stored per project or globally — used for signing deliverables and direct email contact
  • If there are questions, the admin can email the engineer directly from the stored email
  • No login, no profile page, no assignment flow — just a data field

Key Actions from Back Office

  • Add Company — wizard: Company Info → Contract Signer Info → Signing a Contract
  • Invite Company — send email invite for a company to become a Norveo client
  • Create Project — wizard: Select Company → Project Info → Confirm

Empty States

First-time admin: When no companies or projects exist yet, each section shows a guided empty state with a clear CTA. For example, the Companies section shows: “No companies yet. Add your first company to get started.”

04 Layer 2: Detail Views

Clicking an item from a list opens its detail view. Cross-links connect related entities.

Company Detail

Full view of a single company. Cross-links to projects for that company.

Overview Contacts Projects
  • Overview: company name, address, contract status, contract signer, onboarding date
  • Contacts: client-side people (project managers, signers). Add/invite contacts here.
  • Projects: list of all projects for this company (cross-link to Project Detail)

Project Detail

The admin’s project dashboard. From here they can review status or open the full Workspace.

Overview Deliverables Issues Client Project Files Configurator Engineering Options Collaborators BOM
  • Overview: project name, client (cross-link), status, config progress, signed engineer (name + email), key dates
  • Collaborators: client contacts for this project. Signed engineer displayed here.
  • Primary CTA: “Open in Workspace” → launches Layer 3

8 tabs. Billing tab removed (handled in HubSpot).

05 Layer 3: Workspace

The admin opens a project and gets the full CAD tool. Since admin = engineer for now, this is full edit access with the same capabilities.

Admin Intent in Workspace

The admin is the engineer. They enter the Workspace to:

  • Design — create and edit pool geometry, architecture, fixtures, plumbing
  • Engineer — run hydraulic calculations, compliance checks, equipment sizing
  • Deliver — generate plans, specs, permits, and export for the customer

Workspace Tabs

7 tabs, consistent with the Engineer workspace:

Design · Summary · Engineering · Estimate · BOM · Deliverables · History

Catalogue Access

Admin can browse the master parts catalogue from within the workspace via the Components panel in the side panel. Same experience as the engineer.

Navigation Back

  • Breadcrumb: ← Back Office / Projects / Project Name in the top bar
  • Clicking “Back Office” returns to the Back Office; clicking “Projects” returns to the list; clicking the project name returns to Project Detail
Note: Workspace screens are shared with the Engineer role. Since admin = engineer for now, there are no admin-specific differences. If the roles are separated in the future, admin-specific permissions (approval actions, read-only mode) can be layered on.

06 Key Flows

Flows the admin triggers from the back office.

Flow A — Add Company

Triggered from: Companies section.

  1. Company Info Company name, address, industry, size. Validation: name is required, no duplicate company names.
  2. Contract Signer Info Name, email, phone of the person who will sign the contract. Validation: valid email format required.
  3. Signing a Contract Contract review and signature capture (e-signature or upload). Validation: contract must be signed or marked as pending to proceed.

Error / Abandonment Handling

  • Validation failure: inline field-level errors. Admin cannot proceed to next step until resolved.
  • Abandonment: if the admin closes the wizard mid-step, a draft company record is saved. The admin can resume from the Companies list (draft shown with “Incomplete” badge).
  • Duplicate company: if the company name already exists, show a warning with a link to the existing company.

Alternative: Invite company to be client — sends an email invitation. The company accepts and completes their own info, then the admin reviews and approves.

Flow B — Create Project

Triggered from: Projects section or from Company Detail → Projects tab.

  1. Select Company Choose the client company this project belongs to. If triggered from Company Detail, this step is pre-filled. If no companies exist, redirect to Add Company first.
  2. Project Info Project name, address, project type (Residential, Commercial, Spa, etc.). Signed engineer name and email (optional — can be added later from Project Detail). Validation: name and type are required.
  3. Confirm Review summary of company and project info. Confirm to create. Project starts in Draft status.

Error / Abandonment Handling

  • No company exists: show an inline prompt to add a company first, then return to this flow.
  • Abandonment: draft project is not saved until the admin clicks “Confirm”. Closing the wizard discards the draft.

Flow C — Signed Engineer (Data Entry)

Not a flow — just a field. The admin stores the signing engineer’s name and email in one of two places:

  • During Create Project (optional field in the Project Info step)
  • After creation, from the Project Detail → Overview or Collaborators tab

The stored email can be used to email the engineer directly if questions arise. The engineer does not log in to Norveo and does not have a profile.

08 Edge Cases & Error States

Documented edge cases and expected behavior.

Scenario Expected Behavior
First-time admin — everything is empty Each section shows a guided empty state with a clear CTA (e.g. “Add your first company”).
Admin abandons Add Company wizard mid-step Draft company record is saved. Appears in the Companies list with an “Incomplete” badge.
Admin abandons Create Project wizard mid-step No draft is saved until Confirm. Closing the wizard discards the draft.
Admin tries to create a project but no companies exist Redirect to Add Company flow with a message: “You need at least one company before creating a project.”
Duplicate company name during Add Company Inline warning: “A company with this name already exists.” Link to the existing company.
Company invite is not accepted Company remains in “Pending” status. Admin can resend invite or cancel it.
Admin tries to delete a company with active projects Blocked. Message: “This company has active projects. Archive or reassign them first.”
Admin opens workspace for a project with no design yet Show template gallery with “Start Blank” option (same as engineer experience).
Shared login credentials (supervisor shares with staff) Not blocked for now since there is no per-seat charging. Monitor usage if this becomes an issue.
User tries to access admin URL without admin role Redirect to login. After login, route to role-appropriate landing page.
